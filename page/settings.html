Radio Scheduler 設定

設定

ツールを使用するためには、以下のソフトウェアを事前にインストールさせておく必要があります。

  • 音声再生プレイヤー (mpvを使用することを前提に作っています)
  • Python ライブラリー (python3-pip)
  • streamlink
  • streamlink radiko.py

インストールしたソフトウェア情報を入力してください。

which streamlink で表示されたパスを設定してください。
※仮想環境にインストールした場合は、その中で確認してください。

Streamlink プラグイン radiko.py が収納されたディレクトリパスを設定してください。
※ファイル名ではなくディレクトリパスになります。

ラジオや音声ファイルの再生に使用するソフト名を入力してください。

which mpv で表示されたパスを設定してください。

which amixer で表示されたパスを設定してください。

cron スケジュールやコントロールパネルから再起動を行う場合は which reboot で表示されたパスを設定し、sudo visudo から www-data ALL=(root) NOPASSWD: {reboot path} を追加してください。

コントロールパネルからシャットダウンを行う場合は which shutdown で表示されたパスを設定し、sudo visudo から www-data ALL=(root) NOPASSWD: {shutdown path} を追加してください。

音声再生に使用するデバイスを指定してください。音声を出力したいスピーカー機器から音が出ない場合は調整を行ってください。

プレイヤー実行時に使用したいオプションがある場合は使用してください。

Asia/Tokyo 以外のタイムゾーンで利用する場合は変更してください。

各設定情報を入力してください。

時計・天気予報設定

時計の表示や天気予報 (OpenWeathermap) の設定はこちらです。

天気予報情報を取得したい場合は OpenWeather に登録して、取得した API Key を入力してください。※APIへのアクセスは、30分以内は保存されたキャッシュから再利用しています。自動更新は1時間に一回行われます。

天気予報を取得したい位置の緯度を入力してください。
座標は GoogleMap から取得可能です。※マップ上を右クリックで取得できます。モバイル端末なら長押しで表示されます。

天気予報を取得したい位置の経度を入力してください。
座標は GoogleMap から取得可能です。※マップ上を右クリックで取得できます。モバイル端末なら長押しで表示されます。

時計 / 天気予報の表示タイプを選択してください。
Fill: 日付・時計・天気予報 全て表示
Medium: 時計表示・日付と天気予報は簡易表示
Minimum: 時計のみ表示

日付 / 時計 / 天気予報 の文字サイズを変更可能です。
表示させているモニターで文字がはみ出る時や、表示を小さくしたい場合は調整してください。
50% ～ 150% ※デフォルト: 100

時計表示の透明度です。夜間眩しい場合は輝度調整替わりに使用してください。
0% (透明) ～ 100% (不透明) ※デフォルト: 100

各設定情報を入力してください。

メール送信設定

メール送信設定を行う事で cron 実行エラー時に通知される様になります。

エラー発生時に受信したいメールアドレスを設定してください。

メール送信者のアドレスを設定してください。
※送信サーバーと同じドメインで、かつ有効なメールアドレスを設定することを推奨します。

メール送信者名を設定してください。

システム環境に Postfix などのメールサーバーがインストール・設定されている必要があります。
※受信側のスパム対策により、通知メールが届かない場合があります。確実なメール通知を希望する場合は、認証済み外部サーバーを経由可能な PHPMailer や Gmail API を使用してください。

PHPMailer 公式ライブラリを以下のディレクトリへ設置し、ご利用中のSMTPサーバー接続に必要な諸情報を設定してください。
[例] PHPMailer.php が以下の構成となる様にファイルを設置してください。
/{ らぢ助 root ディレクトリ }/send_mail/php-mailer/src/PHPMailer.php

お使いのメール送信サーバーを設定してください。

SMTP認証が必要な場合はチェックを入れてください。

SMTP認証用ユーザー名を設定してください。

SMTP認証用パスワードを設定してください。

SMTPサーバーの暗号化セキュリティ方式を選択してください。

SMTPサーバーのポート番号を設定してください。

Gmail API は、固定のパブリックドメイン ( https://example.com など ) または localhost 環境でのみ認証が可能です。システムをローカルIP ( 192.168.x.x ) 環境で運用する場合は、Gmail API 側からリダイレクトURLの登録ができません。
実機側の localhost 経由で認証を行う場合は、以下のURLを Gmail API のリダイレクトURLへ追加した後に認証を行ってください。
gmail redirect url

ご利用中の Gmail API クライアントID を設定してください。

ご利用中の Gmail API クライアントシークレットを設定してください。

各設定情報を入力してください。

PHP 経由で音声再生が可能な状態か確認しましょう。

音声再生権限確認ボタンを押してください。

再生テスト

Software 設定が完了したら、各ボタンで音声が再生可能か確認してみましょう。
再生されない場合は、ソフト名やパスを間違えているか、音声再生権限が無い可能性があります。

ラジオ及び音声ファイルの再生ボタンを押してください。

Cron

スケジューラーを使用する場合は、以下の cron を追加してください。
sudo crontab -u www-data -e

*/1 * * * * php {PROJECT_ROOT}/php/cron.php
